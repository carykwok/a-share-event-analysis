# A股热点事件行情解析

## 何时触发

当用户要求对以下任一情形做 A 股行情解读时启动：

- 货币/财政政策变动（降息降准、国债增发、赤字率调整）
- 产业政策出台（新兴产业规划、传统产业救市）
- 行业监管事件（强监管打压、资本市场制度利好）
- 地缘政治变化（风险升级或缓和）
- 国际贸易摩擦（关税加征、制裁、豁免、协定）
- 重大政治会议（两会、政治局会议、美联储决议）
- 科技重大突破（AI/芯片、新能源/生物医药）
- 突发黑天鹅（自然灾害、公共卫生）
- 海外市场联动（美股暴跌、人民币汇率异动）
- 大宗商品异动（原油、工业/贵金属）

## 工作流

### Step 1 — 事件分类

读取 `references/templates.md`，根据用户输入的事件描述，从 10 大类 20 子类中**选定最匹配的 1 个子类型**。不确定或跨类时，向用户确认后再继续。

### Step 2 — 联网搜索取数（不向用户索要）

**默认不向用户索要任何背景信息**，通过 WebSearch 自行补齐三要素：

1. **核心事件**：政策/新闻原文关键表述、数据、发布主体、时间
2. **市场反应**：当日主要指数、相关板块涨跌、典型个股、成交异动
3. **撰写时间**：使用今日日期（通过 system context 获取）

**信息源优先级**（严格按此顺序采信）：

- **一级权威**（优先采信）：新华社、人民日报、央行/财政部/证监会/发改委官网、证券时报、中国证券报、上海证券报、证券日报、国务院新闻办
- **二级权威**（可采信）：财新、第一财经、21 世纪经济报道、经济观察报、彭博中文、路透中文
- **三级门户**（作为补充，需交叉验证）：东方财富网、新浪财经、凤凰财经、同花顺、雪球
- **禁用**：自媒体、知乎专栏、不具名股吧评论、无署名小号公众号

**搜索策略**：
- 先搜事件主关键词 + 年份（拿到核心事实）
- 再搜"当日行情 / 板块表现 / 成交量 / 龙头股涨跌"（拿到市场反应）
- 遇相互矛盾说法，以一级权威为准；涉及数据须标注来源
- 如搜索结果仅为"传闻 / 市场预期"而无公告，稿件中必须明确标注"市场传闻，以监管公告为准"

**例外**：用户主动补充/修正信息时，以用户输入为最高优先级。

### Step 3 — 按模板生成

严格执行选定子类型对应的 Prompt（见 `references/templates.md`），产出结构化报告：

- **主标题**（≤15 字）
- **副标题**（20–35 字）
- **正文**（按模板规定字数与分段结构，每段字数符合要求）
- **风险提示**

### Step 4 — 合规红线（对客端强制，不可突破）

**三条绝对禁止**：

1. **禁止写具体股票名称+代码组合**：不出现"中国卫星(600118)"、"宁德时代(300750)"这类标的指引。可在行业链条/产业环节层面描述（如"谐波减速器环节"、"油气开采龙头"），不落到具体公司+代码。
2. **禁止对行情未来走势做绝对性判断**：不用"必然上涨""见底""反弹""突破""翻倍""必定""一定""注定""见顶"等确定性断言。
3. **禁止投资操作建议**：不用"买入""建仓""加仓""减持""超配""低配""规避""止损""建议配置""应对策略"等操作指令。

**可以做的表述**：

- **板块/行业机会提示**：用"相关行业板块可关注"、"值得留意的产业环节"、"投资者关注度或提升"
- **缓和性趋势表述**：一律使用 **"可能 / 或 / 有望 / 预计 / 或将 / 料 / 大概率 / 倾向于 / 存在…可能性"**；历史规律描述改为"历史上类似情形中……"
- **风险提示**：可描述风险因素客观存在，不带"一定会"

**违规词 → 合规改写对照**：

| 违规 | 合规替代 |
|------|---------|
| 超配 / 低配 | 关注 / 留意 |
| 建议配置 / 应对策略 | 关注方向 / 观察角度 |
| 买入 / 建仓 | ——（删除，改为"相关板块可关注"） |
| 规避 / 减持 | 需留意……压力 / 风险敞口较大 |
| 必然/一定/见底/反转/突破 | 可能 / 或将 / 有望 / 存在……可能 |
| 拐点将至 | 或现边际变化信号 |
| 行情修复 | 情绪或呈修复 |
| 个股代码 `(XXXXXX)` | 直接删除代码，或改为"某细分环节龙头" |

### Step 5 — 其他交付约束

- 语言简练专业，每个观点有逻辑支撑，禁止空泛
- 区分"政策受益 vs 业绩受益"、"情绪反弹 vs 趋势反转"、"炒作 vs 产业趋势"
- 涉及量化的给具体数据或敏感度（如"油价每涨10美元，某行业利润影响约X%"），但不以"因此 X 公司股价将涨 Y%"形式给出
- 黑天鹅类保持人文关怀，不以"寻找机会"的语气开篇
- 地缘/监管类保持政治中立，聚焦市场影响
- **文末附 Sources 区**：列出所用一/二级权威信息源的 markdown 链接，便于复核
- **文末附免责声明**：固定一句"本文仅为信息梳理与行业观察，不构成任何投资建议，市场有风险，决策需谨慎。"

## 跨平台适配（Codex / Copilot CLI / 其他）

本 skill 的核心工具是 **Web 搜索 + 本地读写**，对宿主平台的耦合度低。适配清单：

| Claude Code 工具 | Codex 等价 | Copilot CLI 等价 | 说明 |
|---|---|---|---|
| `WebSearch` | `shell` + `curl`/`lynx` 或平台原生搜索工具 | `web_search` | 取得一级权威源正文 |
| `Read` | `shell` + `cat` 或原生 `read_file` | `read_file` | 读 templates.md |
| `Edit` / `Write` | `apply_patch` / `shell` + `sed` | `edit_file` | 如需迭代模板 |

Codex 使用本 skill 的最小方式：

```
<你的 Codex 提示词>
请按照 /skills/a-share-event-analysis/SKILL.md 的工作流处理事件：{事件名}
可用工具：shell（curl 访问公开新闻）、read_file、write_file。
合规红线：不写股票代码 / 不做走势绝对判断 / 不给操作建议。
```

**免责**：无论在哪个平台运行，生成稿件必须附免责声明，合规红线优先级高于风格要求。
